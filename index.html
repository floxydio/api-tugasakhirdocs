API Documentation Update -> 16 Mei

Sign In - Endpoint

Endpoint used for user login.

POST /v1/sign-in

Request

Content-Type: application/json

{
  "username": "value1",
  "password": "value2"
}

Response

{
  "status": 200,
  "accessToken": token,
  "message": "Successfully Login"
}

Sign Up - Endpoint

Endpoint used for user registration.

POST /v1/sign-up

Request

Content-Type: application/json

{
  "nama": "value1",
  "username": "value2",
  "password": "value3",
  "userAgent": "value4"
}

Response

{
  "message": "Successfully Register"
}

Refresh Token - Endpoint

Endpoint used to check or retrieve data from the token.

GET /v1/refresh-token

Request Header

{
  "x-access-token": "value1"
}

Response

{
  "status": 200,
  "data": {}
}

Create Guru - Endpoint

Endpoint used to add a new teacher's data.

POST /v1/guru

Request

{
  "nama": "value1",
  "umur": "value2",
  "mengajar": "value3",
  "status_guru": "value4",
  "rating": "value5"
}

Response

{
  "message": "Successfully Create Guru"
}
